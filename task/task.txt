Data:
 queries.csv          — 400 real banking support queries
                        columns: query_id, text
 
 duplicate_pairs.csv  — for evaluation only, do not use during
                        building. 60 pairs of queries that
                        describe the same issue written
                        differently, labelled easy / medium / hard
 
Task:
Build a pipeline with three phases:
 
1. Duplicate detection (classical methods, DL, etc)
  Find pairs of queries in queries.csv that describe the same
  underlying issue.
 
2. Clustering
  Group the unique queries by issue type. Choose the number of
  clusters yourself and justify it.
 
3. Labelling and ops report
  Use an LLM to label each cluster and generate a short weekly
  ops report: top issues by volume, anything that needs
  escalation, and two recommendations for the support team.
 
Required metrics:
 
           Phase 1 — precision, recall, F1 against
                     duplicate_pairs.csv overall and split by
                     easy / medium / hard. Analyse at least 3
                     pairs your system missed and explain why.
 
           Phase 2 — compare silhouette scores across at least two values of k and explain your final choice. What % of the ground-truth duplicate pairs end up in the same cluster?
 
           Phase 3 — total LLM call count and estimated cost.
 
 README    architecture decisions and tradeoffs, eval results,
           what you would improve with more time.
